# 兵力因子公式初步分析

日期：2026-06-23（更新自 2026-06-22）

## 当前结论

基于正向公式整体拟合结果，当前采用的兵力因子为：

```text
F_source(N) = 1                         , N >= 9000
F_source(N) = (N / 9000) ^ 0.33          , N < 9000
```

这里 `N` 是造成该次伤害时攻击方的当前兵力。指数 `0.33` 是在正向公式中与其他乘区联合优化后的值，单独拟合兵力因子时 `0.38` 的 RMSE 略低，但代入完整公式后 `0.33` 的整体残差更稳定。

## 取数口径

- 唯一数据源：SQLite 的 `damage_contexts`、`participants`、`reports`。
- 攻击方当前兵力：优先读 `damage_contexts.source_context_json.hp`；为空时用该武将 `participants.initial_troops` 补齐。
- 目标当前兵力：使用 `damage_contexts.target_hp_before`；为空时从目标上下文/初始兵力补齐。
- 拟合样本只保留非致死伤害、双方兵力已知、且本次动作没有会心/奇谋等触发的记录。
- 分组时固定 report、攻方、目标、战法、动作类型、兵种、阵营、实时属性、激活 buff 等，尽量让组内只剩兵力变化。
- 为估计攻击方兵力因子，默认只使用目标当前兵力 `>= cap` 的重复组，避免目标低兵力可能带来的承伤项干扰。

## 关于 0.5441 线索

知识表 `game_basic_rules` 中已有一条旧线索：`n^0.5441`，并说明 `n` 应理解为 `当前兵力 / 10000`，且 `9000` 以上无衰减。

当前数据库重复组不支持直接采用 `0.5441`：

- 在默认保守样本中，`alpha=0.5441` 的 `rmse_log=0.015602`，约为 `alpha=0.33` 的 1.75 倍。
- 如果用 `N/10000` 且 `9000` 以上截断，函数在 9000 附近存在跳变；当前拟合更适合写作 `(N / cap)^alpha`，其中 `cap` 约为 `9000`。

因此 `0.5441` 暂时保留为待复核线索，不作为当前公式结论。

## 防守方兵力因子

当前样本不能可靠确认目标当前兵力是否独立影响承伤。默认口径下尝试拟合目标兵力因子：

```text
target factor: rows=10, groups=5, alpha=-0.232781, rmse_log=0.072165, r2=0.128707
```

这个结果样本少、方向不稳定、解释力弱，暂不纳入公式。

## 后续验证方向

1. 优先补充同一攻防双方、同一战法、同一实时属性、目标兵力高于 9000、攻击方兵力跨多个区间的样本。
2. 单独设计普通攻击样本，因为当前普通攻击可用重复组只有 2 组。
3. 拆分战法的隐藏伤害段位，尤其是 `纵马横枪`、`文治武功`、`七进七出`、`千里突袭`。
4. 继续验证无衰减阈值到底是 `9000`、`10000`，还是依初始兵力/等级变化。
