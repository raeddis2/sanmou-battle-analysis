# 新抓取器使用说明

## 常用命令

```powershell
python scripts\capture_battle.py
```

默认流程：

1. 连接游戏进程 `com.bilibili.nslg`。
2. 抓取 xLua 战报字符串到 `data/raw_captures/battle_YYYYMMDD_HHMMSS.txt`。
3. 生成人工检查 Markdown：`reports/battle_YYYYMMDD_HHMMSS.md`。
4. 从流水自动识别 NPC 证据：初始兵力 `16000`，或任意时刻兵力大于玩家上限 `11000`。
5. 从流水自动识别韬略，例如 `《大破》手抄`、`《相思文赋》`、`《守土》善本`。
6. 在 PowerShell 中询问武将红度、金印/品级/自带战法等级、兵种、战法红度等短字段；已识别为 NPC 的字段会自动置 0。
7. 按“战法名 + 红度”从 SQLite 的 `game_skill_versions` 自动取战法说明。
8. 自动更新 Markdown，并写入 SQLite：`data/sanmou_battles.sqlite`。

## 操作方式

启动命令后，在游戏中打开一份战报，等流水加载完。

- 按 `Enter`：保存并解析当前战报，然后自动开启下一份。
- 按 `Ctrl+C`：退出，并尝试解析最后一份当前会话。

## 新战报交互补全

如果是全新的队伍/配置，抓取器只能生成流水，不能知道截图里的红度、金印/品级/自带战法等级、战法红度。它会直接在 PowerShell 里问你，例如：

```text
阵营 A / 赵云
初始兵力:
武将红度 [0]: 5
金印数/品级/自带战法等级 [0]: 1
兵种（默认推测）[骑兵]:
自带战法 [七进七出]:
自带战法等级/红度 [1]: 1
识别到战法：七进七出 / 普攻
本武将战法顺序（用 / 分隔，留空用识别结果） [七进七出 / 普攻]:
【七进七出】几红 [1红]:

本武将配置预览：
- 阵营 A / 赵云: 初始兵力=11000，红度=5红，品级=1，金印=1印
确认本武将配置 (y/n) [Y]:
```

如果流水已经能识别韬略，会显示为：

```text
韬略已从流水识别：《大破》手抄、《相思文赋》、《守土》善本
```

如果流水已经能识别 NPC，会显示为：

```text
阵营 A / 黄月英
初始兵力已从流水识别：16000
初始兵力为 16000，按 NPC 处理：红度/品级/金印/自带战法等级均固定为 0。
韬略按 NPC 规则固定为：无韬略
```

如果初始兵力无法直接确认，但流水里出现了超过玩家上限的兵力，会显示为：

```text
阵营 A / 黄盖
流水中最高兵力为 15631，超过玩家上限 11000，按 NPC 处理：红度/品级/金印/自带战法等级均固定为 0。
韬略按 NPC 规则固定为：无韬略
```

你可以输入短值：

- 红度：直接输入 `0`、`3`、`5` 即可，程序会转成 `0红`、`3红`、`5红`。
- 金印数/品级/自带战法等级：这三项在三谋里按同一个数字记录，直接输入 `0`、`1`、`2` 即可；程序会保存为品级 `N`、金印 `N印`、自带战法红度/等级 `N红`。
- 初始兵力：程序会优先从流水识别；如果仍需人工填写，填 `16000` 会判定该武将是 NPC。即使初始兵力没识别到，只要流水中任意时刻兵力大于 `11000`，也会判定为 NPC，并把武将红度、品级、金印数、自带战法红度、所有战法红度固定为 `0`，韬略固定为 `无韬略`。
- 兵种：填 `盾兵`、`弓兵`、`枪兵`、`骑兵`。`兵种（默认推测）[弓兵]` 表示程序按武将名给出的默认值，不代表已经从流水识别。
- 战法顺序：用 `/` 分隔，例如 `七进七出 / 万人之敌 / 普攻`。
- 韬略：程序会优先从流水中的 `获得战法【《...》】` 自动识别；识别不到时再留给人工补全。

如果输错了：

- 任意输入项可以输入 `重填`、`:redo` 或 `r`，重新填写当前武将。
- 每个武将结束后会显示配置预览；确认时输入 `n`，也会重新填写当前武将。

导入时，武将配置会进入 `participants` 表，战法详情会进入 `report_skill_details` 表。战法详情只记录战法红度、类型、适用兵种和说明；战法没有单独“品级”，旧库里的 `report_skill_details.grade` 列已弃用。

如果你想跳过问答，只生成 MD：

```powershell
python scripts\capture_battle.py --no-review --no-db
```

之后也可以手动编辑 MD，再导入：

```powershell
python scripts\import_reviewed_report.py reports\battle_YYYYMMDD_HHMMSS.md
```

## 复用上一份配置

如果这份战报配置和上一份一样，用：

```powershell
python scripts\capture_battle.py --inherit-config
```

## 常用参数

- `--list-processes`：查看匹配到的游戏进程。
- `--process 关键词`：指定进程关键字，默认 `com.bilibili.nslg`。
- `--pid 12345`：直接指定进程 PID。
- `--inherit-config`：沿用上一份 `reports/battle_*.md` 的人工配置。
- `--previous-config reports\battle_xxx.md`：指定配置来源。
- `--no-review`：不在 PowerShell 问答补全，只生成 Markdown。
- `--no-db`：不自动写 SQLite。
- `--no-parse`：只保存原始捕获文本。
- `--all-chinese`：抓所有中文字符串，用于排查关键词漏抓。

## 数据边界

- SQLite 是当前项目事实源。
- `reports/*.md` 只供人工检查和配置补全。
- `data/raw_captures/*.txt` 是导入 SQLite 的原始捕获来源。
